## Overview

Pocket No is an iOS-first mobile app for fast, low-friction refusal. The current product goes beyond a single quick-copy screen: it gives you a home flow for instant lines, a browse tab for exploring a larger library, a favorites tab for saving the best ones, and settings for appearance and app support surfaces. Apple Intelligence personalization remains a key differentiator on supported iPhones, letting users generate a sharper refusal from typed or voice context without losing the app's punchy tone. Android and web support the core experience, while iOS gets the deepest native integrations through Home Screen Quick Actions and App Intent shortcuts.

### How It Works

1. Open the main app or jump straight into a fast copy flow from a Home Screen Quick Action or Siri Shortcut on iOS.
2. Pocket No fetches a fresh refusal from its server-side catalog, with a smaller local fallback when the network is unavailable.
3. From the main tabs, users can browse more than 1,000 lines by category, search the library, and save favorites for reuse later.
4. If the default line is too generic, the Personalize flow accepts typed or voice context on supported iPhones and uses Apple Intelligence to generate a more specific refusal.
5. The final line appears over an animated dot-grid background, ready to copy with haptics, save to favorites, or replace with another option.

### Key Features

- Instant refusal generation and clipboard copy with haptic feedback
- Browse and search across a 1,000+ line refusal library
- Favorites and recent-history flows stored on-device
- Apple Intelligence personalization on supported iPhones from typed or voice input
- Animated visual treatment powered by a custom Skia shader and matrix-style transitions
- Native iOS entry points through Home Screen Quick Actions and an App Intent / Siri Shortcut target
- Server-backed `GET /api/no` route with automatic fallback to a local catalog
- Deep linking via the `pocketno://` custom URL scheme
- System, light, and dark appearance options
- Support and privacy surfaces for a more complete product shell

### Tech Stack

- Expo 55 / React Native 0.83 / React 19
- Expo Router (typed file-system routing and native tabs)
- Uniwind (Tailwind CSS v4 for React Native)
- React Native Reanimated 4
- @shopify/react-native-skia (custom SKSL shader)
- `@react-native-ai/apple` for Apple Intelligence availability, generation, and voice transcription
- expo-quick-actions & @bacons/apple-targets (iOS native surfaces)
- expo-haptics, expo-clipboard, and Expo API routes
